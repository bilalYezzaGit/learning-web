## 6. PROBLEMES BLOQUANTS (P0)

### ~~BLOCKER #1 — Middleware parcours incoherent~~ FIXE 2026-02-09

**Fichier :** `src/middleware.ts:8`

**Probleme :** Le middleware acceptait `['3eme-math', '2nde-math', '1ere-math', '1ere-tc', 'term-math']` mais seul `1ere-tc` a du contenu.

**Fix applique :** Import dynamique de `VALID_PARCOURS_SLUGS` depuis `@/lib/parcours/config`. Le middleware n'autorise plus que les parcours marques `available: true`. Les parcours sans contenu redirigent vers `/`.

### BLOCKER #2 — Login redirect type mismatch

**Fichier :** `src/components/login-form.tsx:46`

**Probleme :** Code lit `userDoc.data()?.parcours?.slug` mais `UserProfile` type n'a pas de champ `parcours`. Tous les users sont rediriges vers `/select-parcours`.

**Fix :** Soit :
1. Ajouter `parcours?: { slug: string }` au UserProfile type
2. Ou changer la logique pour lire `levelSlug` + `sectionSlug`

### BLOCKER #3 — Firebase security rules non-verifiees

**Impact :** Donnees users potentiellement exposees ou modifiables.

**Action :** Audit obligatoire des regles Firestore avant production :
1. Empecher lecture/ecriture non-auth dans `users/{uid}`
2. Valider ownership sur `activityProgress` subcollection
3. Ne pas exposer de donnees sensibles

### ~~BLOCKER #4 — ANTHROPIC_API_KEY exposure risk~~ VERIFIE 2026-02-09

**Resultat :** Aucun risque. Le SDK `@anthropic-ai/sdk` est utilise UNIQUEMENT dans `src/app/api/scan/route.ts` (API route server-side). La cle `ANTHROPIC_API_KEY` est dans `.env.local` (gitignored) sans prefixe `NEXT_PUBLIC_` — jamais exposee cote client.

### BLOCKER #5 — Content integrity non validee

**Impact :** Atoms references dans molecules peuvent ne pas exister → crash runtime.

**Action :** Script CI validation :
```typescript
const allRefs = getReferencedAtomIds()
for (const id of allRefs) {
  if (!atomExists(id)) throw new Error(`Missing atom: ${id}`)
}
```
